# TranscriptToIssues Quick Reference

## Command

```
/transcript-to-issues <notion-url>
```

## What It Does

1. Fetches Notion page content (meeting transcript)
2. Extracts actionable requirements using AI
3. Searches codebase for related patterns (optional)
4. Generates markdown draft files in `docs/issues-draft/`
5. You review drafts and create GitHub issues manually

## Usage Examples

### Basic Usage

```
/transcript-to-issues https://www.notion.so/Job-Master-Automation-2fdb9debba3381b3b7fcea2c31bb318a
```

Output:
```
✅ Extracted 3 requirements from transcript
✅ Created draft files:
   - docs/issues-draft/2026-02-04-add-lightpath-support.md
   - docs/issues-draft/2026-02-04-automate-attachment-sync.md
   - docs/issues-draft/2026-02-04-implement-stones-river.md
```

### Create GitHub Issue from Draft

```bash
# Review the draft first
cat docs/issues-draft/2026-02-04-add-lightpath-support.md

# Create issue (remember: use --body-file, NOT heredocs!)
gh issue create \
  --title "Add Lightpath Support" \
  --body-file docs/issues-draft/2026-02-04-add-lightpath-support.md \
  --label "enhancement"
```

### Batch Create Multiple Issues

```bash
# Create all drafts as issues
for draft in docs/issues-draft/2026-02-04-*.md; do
  title=$(head -1 "$draft" | sed 's/## Summary//')
  gh issue create \
    --title "$title" \
    --body-file "$draft" \
    --label "enhancement"
done
```

## Draft File Format

Each draft file contains:

```markdown
## Summary

[2-3 sentences describing the requirement]

## Context from Call

> [Relevant discussion snippet from transcript]

## Related Code

- `path/to/file.ts` - Related implementation

## Priority

high | medium | low

## Transcript Source

[Meeting Transcript](https://notion.so/...)

---

**Generated by TranscriptToIssues skill on YYYY-MM-DD**
```

## Troubleshooting

### "Failed to fetch Notion page"

**Causes:**
- Invalid Notion URL
- Missing permissions to page
- Notion MCP not configured

**Fix:**
1. Verify URL is valid
2. Check you can access page in browser
3. Test MCP: `mcp__notion__notion-fetch "https://notion.so/your-page"`

### "No requirements extracted"

**Causes:**
- Transcript has no actionable items (only discussion)
- Very short content
- AI couldn't identify clear decisions

**Fix:**
- Check transcript has decisions or action items
- Add more context to transcript
- Manually create issues for this transcript

### Draft files not created

**Causes:**
- No write access to `docs/issues-draft/`
- Working directory not a project repo

**Fix:**
- Run from project root
- Create directory: `mkdir -p docs/issues-draft`

## Tips

**For best extraction quality:**
- Transcripts should have clear decisions or action items
- Look for phrases like "we should...", "need to fix...", "let's implement..."
- Avoid transcripts that are pure discussion with no conclusions

**Managing drafts:**
- Add `docs/issues-draft/` to `.gitignore` (drafts are ephemeral)
- Delete drafts after creating issues to keep directory clean
- Edit drafts before creating issues to add details

**Integration with planning:**
```
# 1. Extract requirements from transcript
/transcript-to-issues https://notion.so/meeting

# 2. Pick a draft and plan it
/workflows:plan docs/issues-draft/2026-02-04-add-feature.md

# 3. Implement the plan
/workflows:work docs/plans/2026-02-04-feat-add-feature-plan.md
```

## Configuration

### Default Settings

- **Output directory:** `docs/issues-draft/` in current working directory
- **Filename format:** `YYYY-MM-DD-requirement-slug.md`
- **Codebase search:** Top 5 matches per requirement

### Customization

To change defaults, edit:
- `Workflows/NotionTranscriptAnalysis.md` - Workflow logic
- Output directory path (line ~140)
- Search result limit (line ~120)

## Command Reference

### Notion MCP

```bash
# Load Notion tools
ToolSearch({ query: "select:mcp__notion__notion-fetch" })

# Fetch page
mcp__notion__notion_fetch({ url: "https://notion.so/..." })
```

### GitHub CLI

```bash
# Create issue from file
gh issue create --body-file <file> --title "Title" --label "enhancement"

# Create and assign
gh issue create --body-file <file> --assignee "@me"

# Create with milestone
gh issue create --body-file <file> --milestone "Sprint 5"
```
